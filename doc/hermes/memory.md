## 3. 上下文锚点

> 修改 project-context.md 中的约束后，在此记录变更摘要。

### 3.1 业务规则锚点

- **数据来源**：第三方系统通过 JSON API + Bearer Token 推送超声基桩检测数据
- **数据关系链**：**项目(project)** → **基桩(pile)** → **剖面统计(profile_stat)** + **测点(measurement)** → **单桩报告(pile_report)** + **项目报告(project_report)**
- **桩完整性判定**：依据 JGJ 106-2014 标准，通过声速(Vi)和波幅(Ai)参数判定桩身完整性类别（Ⅰ类~Ⅳ类）
- **xlsx 格式参考**：单桩导出参照 `doc/需求/pile1.xlsx` 的 3 个 Sheet
- **写入顺序**：先创建项目 → 再在项目下创建基桩 → 再添加剖面统计和测点数据 → 最后生成报告
- **鉴权机制**：所有业务接口需在 Header 携带 `Authorization: Bearer <token>`
- **日志机制**：每次请求自动记录到文件 + api_log 表，日志表保留 90 天
