- `MeasurementController` - `PUT /api/v1/piles/{pileId}/measurements/{id}` → `POST .../{id}/update`

## 文档更新
- `api-documentation.md` - 所有 PUT 改为 POST，新增日志说明章节

## 所有业务方法统一改为 POST
- 查询列表：`GET /xxx` → `POST /xxx/list`
- 查询详情：`GET /xxx/{id}` → `POST /xxx/{id}`
- 删除：`DELETE /xxx/{id}` → `POST /xxx/{id}/delete`
- 导出：`GET /xxx/{id}/export` → `POST /xxx/{id}/export`
- 查询报告：`GET .../report` → `POST .../report/detail`
- 管理端：`GET/PATCH` → `POST`
- 新增 DTO: `PagedQueryRequest`, `MeasurementQueryRequest`

## 列注释 + Description 属性
- `schema.sqlite.sql` - 重写，每列加中文 SQL 注释，新增 api_key 表
- 所有 8 个 Entity 类 - 每个属性加 `[Description("中文说明")]` + `using System.ComponentModel;`

## 主键 long → Guid 重构
- 所有 8 个 Entity 类：long Id → Guid Id，去掉 [Description] 改用 /// <summary>
- 所有 FK 字段：long → Guid（ProjectId, PileInfoId, PileInfoId）
- 所有 DTO Response：long Id → Guid Id
- 所有 Service 接口及实现：long/Guid 参数调整
- 所有 Controller：long → Guid 参数（ASP.NET 自动转换路由参数）
- schema.sqlite.sql：INTEGER PRIMARY KEY AUTOINCREMENT → TEXT PRIMARY KEY
- _projectId = 1 → _projectId = Guid.NewGuid()（测试文件）
- SeedDataTests：GetAsync→PostAsync, clientId 更新

## [Column] 属性移除 → Fluent 配置
- 所有 8 个实体类：去掉 `[Column(Name = "...")]` 和 `[Column(IsNullable = false)]`，只保留 `[Column(IsPrimary = true)]`
- 新增 `FreeSqlEntityConfig.cs` - Fluent API 集中管理所有列名映射
- Oracle 兼容：只需修改 Fluent 配置里的列名大小写
